# Context: Frontend

- Prefer semantic HTML. Use the right element for the job.
- Keep components small and focused. One component, one responsibility.
- Avoid inline styles. Use utility classes or a design system.
- Accessibility is not optional. Use ARIA labels, keyboard navigation, and sufficient color contrast.
- Minimize client-side JavaScript. Prefer server rendering where possible.
- Handle loading, error, and empty states in every UI component.

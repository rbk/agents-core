# Context: DevOps

- Infrastructure should be defined as code. Avoid manual changes to production.
- Every change should be reviewable and reversible.
- Secrets belong in a secrets manager, not in config files or environment variable plaintext in repos.
- Monitor what matters: availability, latency, error rate, saturation.
- Design for failure. Assume services will go down and plan accordingly.
- Document runbooks for common incidents and deployment procedures.
